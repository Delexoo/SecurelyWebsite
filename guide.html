COMPLETE USER GUIDE

Welcome to the comprehensive guide for Securely. This guide covers every feature, button, and setting in detail to help you get the most out of your security monitoring experience.

GETTING STARTED

First Launch

When you first open Securely, the application will appear as a compact window on the left side of your screen. The program starts in a Disabled state, meaning no monitoring is active until you enable it.

Important: Securely requires Tesseract OCR to be installed for text detection. If Tesseract is not found, you'll see a notification with installation instructions. The program can still run but threat detection will be limited without OCR.

Window Controls

The Securely window has custom controls in the title bar:

  • Minimize Button (−): Minimizes the window to the taskbar
  • Resize Button (⛶): Toggles between compact and enlarged window sizes
  • Close Button (✕): Closes the application (monitoring will stop)

You can drag the window by clicking and holding the title bar or the Securely logo.

DASHBOARD TAB (📊)

Status Label

The status label at the top shows the current monitoring state:

  • ● Disabled: No monitoring is active
  • ● Monitoring: Continuous 24/7 monitoring is active
  • ● Paused: Monitoring is temporarily paused (e.g., on login pages)
  • ● Scanning (Xm remaining): Temporary scan is active with time remaining

Quick Scan Button

Quick Scan

Performs a single, immediate scan of your screen:

  • Takes one screenshot of your primary monitor
  • Extracts text using OCR (if available)
  • Analyzes content with AI for threats
  • Shows results immediately via notification
  • Does NOT enable continuous monitoring
  • Does NOT pause on login pages (pause only works during continuous monitoring)

Use Case: Perfect for quick checks when you want to verify a specific screen or URL without starting continuous monitoring.

Scan for 15m Button

Scan 15m

Starts a temporary monitoring session that runs for 15 minutes:

  • Continuously scans your screen every second
  • Automatically stops after 15 minutes
  • Shows a countdown timer in the status label
  • Can be stopped early by clicking "Turn Off"
  • Will pause automatically if a login or payment page is detected
  • When paused, the temporary scan is disabled and monitoring enters pause mode

Note: If a login or payment page is detected during a temporary scan, the scan will stop and the program will enter pause mode. You'll be notified and monitoring will auto-resume after 3 minutes.

Turn On / Turn Off Button

Turn On / Turn Off

Controls continuous 24/7 monitoring:

  • Turn On: Starts continuous monitoring that runs indefinitely
  • Shows a confirmation dialog the first time you enable it
  • Scans your primary monitor every second
  • Analyzes all visible content for threats
  • Button changes to "Turn Off" when monitoring is active
  • Turn Off: Stops all monitoring immediately

Confirmation Dialog: The first time you click "Turn On", you'll see a confirmation dialog explaining that continuous monitoring will keep Securely running in the background and may analyze portions of your screen. Click "Enable" to proceed or "Cancel" to go back.

SETTINGS TAB (⚙️)

Hitboxes Toggle

When enabled, displays visual boxes around detected threats on your screen:

  • Red boxes appear around suspicious or malicious content
  • Boxes update in real-time (every 0.3 seconds when enabled)
  • One box per unique threat (no stacking)
  • Boxes automatically disappear when threats are no longer detected
  • No text labels - just visual indicators
  • Only works when monitoring is active

Tip: Hitboxes are great for visual feedback, but they may slightly increase CPU usage due to faster update intervals. Disable if you experience performance issues.

More Strict Toggle

Enables enhanced keyword-based threat detection:

  • Adds additional keyword scanning alongside AI analysis
  • Scans for common phishing terms, suspicious URLs, and threat indicators
  • Works in combination with AI analysis for comprehensive protection
  • May increase false positives but provides extra security layer

Notifications Toggle

Controls desktop notifications for threat detections:

  • When enabled: Shows Windows desktop notifications when threats are detected
  • When disabled: Threats are still logged but no notifications appear
  • Notifications work best with Windows 10/11 native toast support
  • Falls back to alternative methods if native support isn't available

Sound Toggle

Enables audio alerts for threat detections:

  • Plays a system beep sound when threats are detected
  • Only works on Windows (uses winsound)
  • Works independently of notifications

Test Notification Button

🚨 Test Notification

Tests your notification system:

  • Sends a test desktop notification
  • If successful, shows a second confirmation notification
  • If notifications aren't working, displays instructions for installing required libraries
  • Use this to verify your notification setup is working correctly

Advanced Section

Test Mode Toggle

Enables debug/test mode for development:

  • Shows additional debug information in console
  • May save training screenshots (if enabled)
  • Primarily for developers and troubleshooting

View Console Log Button

🖥️ View Console Log

Opens a console log viewer window:

  • Shows real-time console output from Securely
  • Useful for debugging and seeing what the program is doing
  • Updates automatically as events occur

AUDIT LOG TAB (📋)

Overview

The Audit Log displays all detected threats and security events in chronological order. Each entry is shown as a card with detailed information.

Quick Scan Button

Quick Scan

Same as the Quick Scan button on the Dashboard - performs an immediate single scan.

Clear Log Button

Clear Log

Clears all entries from the audit log:

  • Removes all threat cards from the display
  • Clears the log file on disk
  • Action cannot be undone

Warning: Clearing the log permanently deletes all threat history. Make sure you've reviewed important entries before clearing.

Threat Cards

Each detected threat appears as a card showing:

  • Timestamp: When the threat was detected
  • Threat Type: Category of the threat (phishing, malware, suspicious URL, etc.)
  • Details: Specific information about what was detected
  • AI Analysis: Full AI response explaining why it was flagged
  • Screenshot: Visual reference of what was on screen (if available)

AUTO-PAUSE FEATURE

How It Works

Securely automatically pauses monitoring when it detects legitimate login or payment pages to protect your privacy:

  • Detects login pages (email, social media, banking, etc.)
  • Detects payment pages (checkout, billing, etc.)
  • Automatically pauses monitoring for 3 minutes
  • Shows a notification explaining the pause
  • Auto-resumes after 3 minutes
  • Status label changes to "● Paused"

When Auto-Pause Activates

  • During 24/7 Monitoring: Pauses automatically, resumes after 3 minutes
  • During Temporary Scan (15m): Stops the temporary scan and enters pause mode
  • During Quick Scan: Does NOT pause (only shows a notification)

Manual Resume

You can manually resume monitoring by:

  • Clicking "Quick Scan" or "Scan 15m" - these will automatically resume monitoring if paused
  • Waiting for the 3-minute auto-resume timer

MONITORING BEHAVIOR

Primary Monitor Only

Securely only monitors your primary monitor:

  • Uses Windows API to detect the primary monitor
  • Ignores secondary monitors completely
  • Window-specific captures only work if the window is entirely on the primary monitor
  • This ensures privacy and focuses monitoring on your main workspace

Scan Interval

Monitoring frequency depends on settings:

  • Normal Mode: Scans every 1 second
  • Hitboxes Enabled: Scans every 0.3 seconds for faster visual updates
  • OCR text extraction runs on every scan

Text Extraction (OCR)

Securely uses Tesseract OCR to extract visible text from your screen:

  • Extracts all visible text from the current screen
  • Text is sent to AI for analysis
  • If no text is detected, AI will flag it as "UNSURE" and recommend manual review
  • OCR is essential for accurate threat detection

Installation: Tesseract OCR must be installed separately. Download from: https://github.com/UB-Mannheim/tesseract/wiki

AI THREAT ANALYSIS

How AI Analysis Works

Securely uses Groq API with advanced language models to analyze screen content:

  • Extracted text is sent to AI for analysis
  • AI models used (in order of preference):
    • meta-llama/llama-4-maverick-17b-128e-instruct
    • meta-llama/llama-4-scout-17b-16e-instruct
    • openai/gpt-oss-120b
    • qwen/qwen3-32b
    • kimi/kimi-k2-0905
  • If one model fails, automatically tries the next
  • AI is trained to be aggressive in threat detection, especially for emails

AI Response Types

The AI can respond with:

  • SAFE: Content appears legitimate and safe
  • THREAT: Suspicious or malicious content detected
  • UNSURE: Cannot determine with confidence (recommends manual review)
  • LEGITIMATE_LOGIN: Detected a legitimate login page (triggers auto-pause)
  • LEGITIMATE_PAYMENT: Detected a legitimate payment page (triggers auto-pause)

Threat Detection Focus

The AI is specifically trained to detect:

  • Phishing emails and websites
  • Malicious URLs and links
  • Suspicious sender addresses
  • Fake login pages
  • Social engineering attempts
  • Malware distribution attempts

TROUBLESHOOTING

Notifications Not Working

If notifications don't appear:

  • Click "Test Notification" to diagnose the issue
  • Install win11toast or win10toast if prompted
  • Check Windows notification settings - ensure notifications are enabled for Securely
  • Try restarting the application

No Text Detected

If you see "No text detected on screen":

  • Verify Tesseract OCR is installed correctly
  • Check that Tesseract is in your system PATH or installed to the default location
  • Restart Securely after installing Tesseract
  • Some screens (images, videos) may not have extractable text

AI Analysis Failing

If AI analysis isn't working:

  • Check your internet connection
  • Verify Groq API key is valid (check console log)
  • API may be experiencing rate limits - wait a few minutes and try again
  • View console log for specific error messages

Performance Issues

If Securely is using too much CPU:

  • Disable Hitboxes (reduces scan frequency)
  • Use Quick Scan instead of continuous monitoring when possible
  • Close other resource-intensive applications
  • Ensure Tesseract OCR is properly installed (missing OCR can cause errors)

BEST PRACTICES

Recommended Settings

  • Notifications: Keep enabled to stay informed of threats
  • Hitboxes: Enable for visual feedback, disable if performance is an issue
  • More Strict: Enable for maximum security, may increase false positives
  • Sound: Optional - useful if you're away from screen

When to Use Each Scan Type

  • Quick Scan: Use for one-time checks, verifying specific URLs or emails
  • Scan 15m: Use for temporary monitoring sessions (browsing, checking emails)
  • Turn On (24/7): Use for continuous protection during extended computer use

Privacy Considerations

  • Securely only monitors your primary monitor
  • Auto-pause protects login and payment pages
  • All processing is done locally (OCR) or via secure API (AI analysis)
  • Threat logs are stored locally on your computer
  • No screen content is stored permanently (unless training mode is enabled)
